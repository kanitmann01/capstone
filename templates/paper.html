Brand Guard: Explainable Brand-Aware Phishing Detection

A hybrid approach combining brand recognition, heuristics, threat intelligence, and machine learning for day-zero phishing triage.

Abstract

Existing phishing detection tools fall into three families: rule-based heuristics (fast but content-blind), threat-intel blocklists (industry-grade but day-zero blind), and generic ML classifiers (pattern-aware but brand-blind). Brand Guard is the first system to combine all three families with explicit brand reasoning and explainability. On a 240-URL held-out test set with 100 banks in the detection inventory, Brand Guard achieves 98% accuracy and 98% recall — the only evaluated system with meaningful day-zero detection capability.

Methodology

Dataset

  • 1,600 URLs total (800 phishing, 800 legitimate)
  • Phishing sources: OpenPhish (300), URLhaus (500)
  • Legitimate sources: Tranco top list filtered to bank/brand/SaaS domains
  • Stratified 70/15/15 split with host-disjoint constraint

Models

  • FastText: URL-only corpus with brand + heuristic tokens
  • TensorFlow: Dense classifier (128→64) on 27 URL features
  • Brand Recognition: FAISS + Bloom + Levenshtein (110 brands)

Key Findings

1. Blocklist Collapse: Netstar lookup scored 65% accuracy on historical data — equivalent to random guessing for day-zero URLs missing from feeds.

2. ML Distribution Mismatch: The public HF BERT classifier collapsed to 50% accuracy, validating that generic ML is brand-blind.

3. Brand Recognition Drives Differentiation: Brand Guard's 98% recall comes from the brand-recognition layer detecting typosquats, homographs, and deceptive subdomains before any feed lists them.

4. Official-Domain Override: Precision of 96% is maintained by short-circuiting legitimate bank login pages to score zero.
